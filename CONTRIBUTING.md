## ОСНОВНЫЕ ПРАВИЛА
### Issue - постановка задачи или обозначение проблемы

1. На каждое нововведение создавать Issue, если еще нет такового
2. Если есть основной Issue, но требуется конкретика по задаче, создается sub-issue
3. Название Issue - краткое описание задачи, в поле ниже более конкретное описание
4. Каждому Issue присваивается приоритет - Priority 1 - важно и срочно Priority 2 - важно, но не срочно Priority 3 - не важно, но срочно Priority 4 - не важно и не срочно

### Commit - фиксация изменений

1. Каждое минимальное изменение должно допровождаться коммитом с указанием номера Issue и кратким описанием что было сделано.

Пример Commit

Неправильно: "Создал автобус"

Правильно: "#28 Создал автобус №1"

2. Коммиты, связанные с Sumo файлами, отправляются ТОЛЬКО в ветку sumo_files. Любые коммиты в main ветку будут отклоняться.
3. НИЧЕГО НЕ ПУШИТЬ В main ВЕТКУ!!!
